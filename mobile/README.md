# FitTrack AI - Mobile App

A professional AI-powered fitness and nutrition tracking mobile app built with React Native (Expo).

## Features

- **AI Food Analysis** - Take a photo of your food and get instant nutritional breakdown powered by Google Gemini AI
- **Water Tracking** - Track daily hydration with quick-add buttons and progress visualization
- **Workout Logging** - Log exercises with sets, reps, and weights
- **Workout Library** - Browse curated workout videos by muscle group and difficulty
- **AI Diet Coach** - Get personalized meal plans generated by AI
- **Health Analytics** - View progress charts, trends, and AI-powered insights
- **Health Calculators** - BMI, BMR, and TDEE calculators
- **Secure Auth** - JWT-based authentication with secure token storage

## Tech Stack

- **Framework**: React Native with Expo SDK 55
- **Navigation**: Expo Router (file-based routing)
- **UI**: Custom components with dark theme, LinearGradient, Ionicons
- **State**: React Context API
- **API**: Axios with interceptors for auth
- **Storage**: expo-secure-store for auth tokens
- **Native**: Camera, Haptics, Image Picker

## Getting Started

### Prerequisites

- Node.js 18+
- Expo CLI: `npm install -g expo-cli`
- EAS CLI: `npm install -g eas-cli`
- Backend server running (see ../backend/)

### Install Dependencies

```bash
cd mobile
npm install
```

### Configure Backend URL

The mobile app reads backend URL from `EXPO_PUBLIC_API_URL`.

1. Create a local env file:

```bash
cp .env.example .env
```

2. Set your backend URL:

```env
EXPO_PUBLIC_API_URL=https://api.yourdomain.com
```

For local physical-device testing, use your machine LAN IP, for example:

```env
EXPO_PUBLIC_API_URL=http://192.168.1.100:8000
```

### Run Development

```bash
# Start Expo development server
npx expo start

# Run on iOS Simulator
npx expo start --ios

# Run on Android Emulator
npx expo start --android
```

## Building for App Store / Play Store

### 1. Login to EAS

```bash
npx eas-cli login
```

### 2. Configure your project

Update `eas.json` submit values with your:
- Apple Developer credentials (iOS)
- Google Play service account (Android)

Update `app.json` identifiers as needed:
- `expo.ios.bundleIdentifier`
- `expo.android.package`
- `expo.extra.eas.projectId` (after `eas init`)

Set the API URL in EAS so cloud builds use your production backend:

```bash
npx eas-cli env:create production --name EXPO_PUBLIC_API_URL --value https://api.yourdomain.com --visibility plaintext --scope project --non-interactive
```

### 3. Build

```bash
# Preview (internal testing)
npx eas-cli build --platform all --profile preview

# Production
npx eas-cli build --platform all --profile production
```

### 4. Submit to stores

```bash
# Submit to App Store
npx eas-cli submit --platform ios

# Submit to Google Play
npx eas-cli submit --platform android
```

To allow free downloads:
- Google Play Console pricing: `Free`
- App Store Connect price: `Tier 0 (Free)`

## App Store Requirements

### iOS (App Store)
- Privacy Policy URL required
- App screenshots (6.7", 6.1", 5.5" iPhones + iPad)
- App description, keywords, categories
- Camera & Photo Library usage descriptions (already configured in app.json)

### Android (Play Store)
- Privacy Policy URL required
- Feature graphic (1024x500)
- Screenshots (phone + 7" tablet + 10" tablet)
- App description, categorization
- Content rating questionnaire

## Environment

The app connects to the backend API. Make sure the backend is running before testing:

```bash

cd backend
uvicorn server:app --host 0.0.0.0 --port 8000 --reload
```
